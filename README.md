# Что это такое и зачем

**Проблема**: при одновременном подключении Amnezia WireGuard (для обхода блокировок) и FortiClient (корпоративный VPN) пропадал интернет — DNS-запросы не проходили.

**Причина**: FortiClient включает pf-правила (macOS файрвол), которые блокировали DNS-трафик к корпоративным серверам 10.54.68.68 / 10.54.129.129, когда пакеты шли через нестандартную маршрутизацию Amnezia.

**Решение**: правила pass out quick для DNS-трафика, которые явно разрешают DNS-пакеты до того, как FortiClient'овский файрвол их заблокирует.

**Артефакты решения:**

pf-dns-redirect.conf — pf-правила
fix-dns.sh — включает правила
restore-dns.sh — выключает правила

**Порядок работы на каждый день**

- Включить Amnezia
- Включить FortiClient
- `~/fix-dns.sh`
- Работать — доступны и корп ресурсы, и многие другие
- Когда закончил: `~/restore-dns.sh`
